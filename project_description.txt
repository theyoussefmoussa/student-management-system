What does the project do? 
Student Management System is a CLI project who more likely for developers and tech people.
has four main classes: Person, Student, Grade, Course each class has its attributes and methods

> v2.0 — Migrated from in-memory storage to a persistent MySQL database using the Repository Pattern.

Person: 
    - Name: Person Name
    - Age: Person Age
    - Gender: Person Gender
    - get_details():  for printing person info

Student:
    Inherit Person: 
        Name: Student Name (At Least 3 Characters)
        Age: Student Age   (In Range 17 and 29 Included)
        Gender: Student Gender (Male or Female) Handle lowercase and uppercase
        student_id: Unique id for each student (checked against database for uniqueness)

Grade: 
    - Course name: course name passed as parameter OR user type it in runtime with validation
    - Grade: Grade passed as parameter OR user type it in runtime with validation
    - Kept for reference — grade data is now stored directly in the database via GradeRepo

Course:
     - Name: Course name
     - Credit: Credit Hours of Course


### service classes: to handle business logic with static methods

CourseService: 
display_course_info(course_name)
    - void function that prints complete course information: name, credit, enrolled_students number
    - displays each student with their id enrolled in this course
    - displays average grade for this course using get_course_average()
    - checks if the course has NO enrolled students

get_course_average(course_id)
    - checks if the course has NO enrolled students
    - collects all grades for this course from all students via GradeRepo
    - returns total grades / grades count
    - check if there is NO grades: print a message for user and return None

StudentService: 
get_grade_for_course(student_id, course_name)
    - gets student grade for a specific course
    - gets course_id from CourseRepo
    - gets grade from GradeRepo
    - if found return the grade value
    - if not print a message for user then return none

enrol_in_course(student_id, course_name)
    - gets course_id from CourseRepo
    - checks if student is already enrolled via EnrollmentRepo
    - Add enrollment record to database
    - Print success message to user

drop_student_from_course(student_id, course_name)
    - gets course_id from CourseRepo
    - checks if student is already enrolled
    - if not print message to user and return
    - remove enrollment record from database
    - remove grade record from database
    - print success message to user

add_grade_to_student(student_id, course_name, grade_value)
    - Validate grade value
    - gets course_id from CourseRepo
    - check if student is enrolled in the course via EnrollmentRepo
    - if not print message to user and return
    - add grade to database via GradeRepo

calculate_average_grade(student_id)
    - gets all grades from GradeRepo
    - handle empty grades
    - calculate average
    - return average

display_full_info(student_id)
    - gets student data from StudentRepo
    - Display Student Personal Data: Name, Age, Gender, Student id
    - Display enrolled courses via EnrollmentRepo
    - Display grades with course names via GradeRepo and CourseRepo
    - display average
    - No Return 

display_student_average(student_id)
    - take average from calculate_average_grade(student_id) function
    - if average is None return
    - Display Grade Letter (A, B, C, D, F)
    - No Return 

update_student_grade(student_id, course_name, new_grade)
    - validate new grade
    - if not returns
    - gets course_id from CourseRepo
    - check if grade exists via GradeRepo
    - if not found print message to user
    - update grade in database


### repository classes: to handle all database operations

StudentRepo:
    add_student(student)
        - INSERT student record into students table
    get_all_students()
        - SELECT all students
    get_student_by_id(student_id)
        - SELECT student WHERE student_id matches
    delete_student(student_id)
        - DELETE student record

CourseRepo:
    add_course(course)
        - INSERT course record into courses table
    get_all_courses()
        - SELECT all courses
    get_course_by_name(course_name)
        - SELECT course WHERE name matches
    get_course_by_id(course_id)
        - SELECT course WHERE course_id matches
    delete_course_by_id(course_id)
        - DELETE course record

EnrollmentRepo:
    enroll_student(student_id, course_id)
        - INSERT enrollment record
    drop_student(student_id, course_id)
        - DELETE enrollment record
    get_courses_for_student(student_id)
        - SELECT all course_ids for a student
    get_students_for_course(course_id)
        - SELECT all student_ids for a course
    is_enrolled(student_id, course_id)
        - SELECT and return True if record exists, False otherwise

GradeRepo:
    add_grade(student_id, course_id, grade_value)
        - INSERT grade record
    get_grade(student_id, course_id)
        - SELECT grade WHERE student and course match
    get_all_grades_for_student(student_id)
        - SELECT all grades for a student
    update_grade(student_id, course_id, new_grade)
        - UPDATE grade record
    delete_grade(student_id, course_id)
        - DELETE grade record

constants (hardcode info)

    - ID_LENGTH = 6
    - MIN_NAME_LENGTH = 3
    - MIN_AGE = 17
    - MAX_AGE = 29
    - MIN_CREDIT = 2
    - MAX_CREDIT = 5
    - MIN_GRADE = 0
    - MAX_GRADE = 100

display: 
    - display_menu()
        - 1. Create Student
        - 2. Create Course
        - 3. Enroll Student in Course
        - 4. Add Grade To Student
        - 5. Display Student Info
        - 6. Display Course Info
        - 7. Calculate Student Average
        - 8. Drop Student from Course
        - 9. Get Grade for Course
        - 10. Update Student Grade
        - 0. Exit

helper: 
    - user_choice(label, choice_collector)
        - checks user input as valid integer
        - convert it to int
        - check range from 0:n-1
        - return index (choice - 1)
        - if not correct print error message
    - pick_student()  
        - fetches all students from StudentRepo
        - displays students by name
        - returns selected student_id
    - pick_course()  
        - fetches all courses from CourseRepo
        - displays courses by name
        - returns selected course_name

validators:
    - get_valid_gender()
        - checks if gender is either male or female 
        - handles lowercases, uppercases and empty input
        - return gender
    
    - get_valid_age()
        - checks if input is a number
        - convert it to int
        - check age range
        - return age

    - get_valid_name()
        - checks empty input 
        - must be at least 3 characters
        - only alphabet allowed
        - title full name 
        - return name

    - get_valid_unique_id()
        - checks empty input
        - checks is numeric
        - must be equal to ID_LENGTH
        - checks uniqueness against database via StudentRepo
        - return id
    
    - get_valid_grade()
        - checks empty input
        - checks only positive numbers allowed
        - convert to int
        - checks range
        - return grade

    - get_valid_credit()
        - checks empty input
        - checks only positive numbers allowed
        - convert to int
        - checks range
        - return credit

    - get_valid_course_name()
        - checks empty input
        - handle spaces
        - lowercase research for matching
        - if matched return course name
        - if not valid print sample of available courses


database folder
    connection.py
        - loads .env credentials using python-dotenv
        - returns a MySQL connection via mysql-connector-python

    schema.sql
        - CREATE DATABASE student_management_system
        - CREATE TABLE students
        - CREATE TABLE courses
        - CREATE TABLE enrollments (many-to-many: students <-> courses)
        - CREATE TABLE grades
        - All foreign keys use ON DELETE CASCADE


main: 
    - initialize StudentRepo and CourseRepo instances
    - displaying menu
    - user choose an action 
    - perform that action using services and repos
    - only runs in main.py
